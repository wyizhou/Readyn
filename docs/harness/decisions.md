# 已确认决策

本文件记录后续实现应默认遵守的决策。若需要改变，先确认并更新本文件。

## 设计与产品

- Open Design `v0.1.0` 是当前最新设计规格。
- UI 语言以中文为主，必要时保留产品名、品牌名和训练指标缩写。
- `index.html` / 首页职责是负荷总览，不是入口卡片页。
- 主导航固定为：
  - `01 总览`
  - `02 活动`
  - `03 健康`
  - `04 连接`
  - `05 教练`
- 健康是唯一允许的二级菜单。
- 健康二级菜单当前只包含：
  - 睡眠
  - 体重
- 活动详情参考 RUNALYZE 的信息密度，但不照搬 RUNALYZE 视觉。
- 当前视觉方向是浅色、专业、高密度数据产品；不回到旧深色 Claude Design 方向。

## 数据边界

- Garmin 连接器只同步活动、睡眠、体重和原始流。
- 不从 Garmin 直接信任最终训练负荷结论。
- 训练负荷由 Trainalyze 平台自算。
- 平台自算指标包括 TRIMP、ATL、CTL、TSB、A:C、Easy TRIMP。
- 活动运动类型筛选必须从活动数据 `sportType` 去重生成。
- 无数据不能伪造，必须显示空值、`—` 或灰色空态。
- 没有 GPS、功率、跑姿、身体电量等数据时，不创建假图表或假指标。

## 实现方式

- Open Design HTML 是高保真规格，不是生产代码。
- 实现应拆成 React 组件、状态、数据结构和测试。
- 优先复用当前项目已有组件、API 和测试结构。
- 设计与现有代码不一致时，不假设现有代码已经符合设计；需要明确迁移范围。

## 交付规则

- 完成产品功能后，需要按项目规则运行 lint 和 testing。
- 完成功能后需要本地 commit 并 push GitHub，除非用户明确要求不要提交。
- 只读分析不需要 commit/push。
